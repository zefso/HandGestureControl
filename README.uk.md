> Керуй комп'ютером жестами рук у реальному часі — без клавіатури та миші.

Побудовано на основі **MediaPipe** (відстеження рук) + **LSTM** нейромережа для розпізнавання жестів, з власним HUD та багатопрофільною системою дій.

## Можливості

- **Відстеження рук у реальному часі** — MediaPipe визначає до 2 рук (21 точка кожна) на 30+ FPS
- **LSTM-розпізнавання жестів** — 2-шаровий LSTM (128 прихованих, dropout 0.3) класифікує 10 динамічних жестів
- **3 режими роботи** — GESTURES, MOUSE, VOLUME, перемикаються жестами під час сесії
- **AirMouse** — права рука керує курсором; ліва прокручує (позиційний скрол) або виділяє текст (кулак)
- **Керування гучністю** — лівий кулак + відстань правої руки регулює системну гучність
- **4 профілі дій** — `default`, `browser`, `media`, `vscode`; **автопереключення** за назвою активного вікна
- **Конфігурація через JSON** — зміни жестів, дій, порогів і профілів без перезапуску
- **HUD** — оверлей режиму/жесту/FPS, панель налаштувань, тест-режим з ймовірностями по класах
- **Розумний колектор даних** — підказки варіативності, перевірка якості, валідація сумісності датасету
- **Автотести** — pytest-набір для AirMouse, VolumeController, keypoints та LSTM-моделі

## Жести

| Жест | Рука | Опис |
|:---|:---:|:---|
| `static` | П | Нейтральна поза — без дії |
| `swipe_right` | П | Свайп правою рукою вправо |
| `swipe_left` | Л | Свайп лівою рукою вліво |
| `ok` | П | Торкнутись вказівний + великий → режим MOUSE |
| `stop` | П | Відкрита долоня → назад до GESTURES |
| `browser` | П | Знак "мир" (вказівний + середній вгору) |
| `fist_left` | Л | Кулак лівою рукою |
| `swipe_up` | П | Свайп правою рукою вгору |
| `swipe_down` | П | Свайп правою рукою вниз |
| `call_me` | П | Мізинець + великий (знак телефону) |

## Режими роботи

### Режим GESTURES (за замовчуванням)
LSTM розпізнає жести та виконує дії з активного профілю.
Буфер підтвердження (3 з 5 кадрів) запобігає випадковим спрацюванням.
Свайпи блокуються після спрацювання до повернення руки до `static`.

### Режим MOUSE
Увімкнути: тримати жест `ok` ~7 кадрів.

| Дія | Як виконати |
|:---|:---|
| Рух курсору | Кінчик вказівного пальця (права рука) |
| Лівий клік | Зсунути великий + вказівний (права рука) |
| Правий клік | Зсунути великий + середній (права рука) |
| Прокрутка | Ліва рука — витягнутий вказівний; Y-позиція відносно центру екрану задає напрям і швидкість |
| Виділення тексту | Лівий кулак + клік правою — курсор рухається при затиснутій ЛКМ |

Вимкнути: тримати жест `stop`.

### Режим VOLUME
Лише в режимі GESTURES. Тримати лівий кулак ~7 кадрів, потім змінювати відстань великий–вказівний правої руки для регулювання гучності.

## Профілі

Профілі прив'язують жести до дій. Активний профіль автоматично змінюється за назвою активного вікна кожні 60 кадрів.

| Профіль | Автоактивується для |
|:---|:---|
| `default` | будь-яке вікно |
| `browser` | Chrome, Firefox, Edge, Opera, Brave |
| `vscode` | VS Code, PyCharm, IntelliJ |
| `media` | Spotify, VLC, YouTube, Netflix |

Ручне перевизначення: клавіші `1`–`4` фіксують конкретний профіль (вимикає автопереключення).

Усі профілі та їх дії описані в `src/gestures.json` — редагуй без перезапуску.

## Використання

### Запуск програми

```bash
python src/main.py
```

Відкриється вікно з зображенням камери та HUD-оверлеєм. Підніми праву руку перед камерою.

### Збір навчальних даних

```bash
# Зібрати всі жести (200 прикладів кожен за замовчуванням)
python src/smart_collector.py

# Зібрати лише один жест
python src/smart_collector.py --target static --count 300

# Очистити старий датасет і почати з нуля
python src/smart_collector.py --clean
```

**Керування в колекторі:** `Space` — записати · `N` — наступний жест · `P` — попередній · `D` — видалити останній · `Q` — вийти

Підказки варіативності змінюються автоматично — виконуй їх, щоб модель краще узагальнювалась (відстань, кут, швидкість, освітлення).

### Навчання моделі

```bash
python src/train.py
```

- Завантажує дані з `data/`, застосовує аугментацію (шум, масштаб, часовий зсув)
- Збалансована вибірка через `WeightedRandomSampler` + `CrossEntropyLoss` з вагами класів
- Early stopping (patience = 20 епох за замовчуванням)
- Найкращі ваги зберігаються в `models/gesture_lstm_best.pth`
- Графіки зберігаються в `diagram/` (крива навчання + матриця плутанини)

### Запуск тестів

```bash
pytest tests/
```

## Конфігурація

Всі параметри знаходяться в `src/gestures.json`:

```jsonc
{
  "gestures": ["static", "swipe_right", ...],    // 10 жестів
  "active_profile": "default",
  "profiles": { ... },                            // 4 профілі з діями для кожного жесту
  "model": {
    "threshold": 0.92,       // мінімальна впевненість для спрацювання
    "seq_length": 30,        // кадрів на послідовність
    "switch_frames": 7,      // кадрів утримання для зміни режиму
    "cooldown_frames": 40,   // кадрів між спрацюваннями жестів
    "swipe_cooldown_frames": 15
  },
  "mouse": {
    "smoothing": 0.3,        // коефіцієнт EMA (0 = нерухомий, 1 = сирий)
    "click_zone_x": [0.25, 0.75],
    "click_zone_y": [0.25, 0.65]
  },
  "volume": {
    "smoothing": 0.15,
    "distance_min": 0.02,
    "distance_max": 0.2
  }
}
```

## Клавіатурні скорочення

| Клавіша | Дія |
|:---|:---|
| `H` | Увімкнути / вимкнути довідку |
| `S` | Відкрити / закрити панель налаштувань |
| `T` | Тест-режим — показує ймовірності по класах, дії не виконуються |
| `P` | Пауза / Продовжити |
| `1` – `4` | Переключити профіль: default / browser / media / vscode |
| `Q` / `Esc` / `M` | Вийти |

## Вирішення проблем

**Руки не визначаються**
Покращи освітлення, уникай контрового світла, тримай руки в кадрі.

**Курсор смикається**
Збільш `mouse.smoothing` в `gestures.json` (наприклад, `0.4`).

**Жести плутаються**
Увімкни тест-режим (`T`) — побачиш смуги впевненості по кожному класу. Збери більше прикладів для проблемного жесту через `smart_collector.py --target <жест>`.

**Свайп спрацьовує кілька разів**
Вже виправлено прапором `_swipe_locked` — свайп повторно спрацьовує лише після повернення руки до `static`. Якщо проблема залишається, збільш `swipe_cooldown_frames` в `gestures.json`.

**`Module not found` при запуску**
Активуй віртуальне середовище: `venv\Scripts\activate`, потім `pip install -r requirements.txt`.
